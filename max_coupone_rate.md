# クーポン推移の最大粗利益追及編

## 1. 目的

この仕様書は、HAYA-TOKU において  
**クーポン割引率の推移を「最大粗利益」を基準に設計するための原本仕様** を定義するものです。

この仕様が表すものは、単なる思いつきの割引率ではなく、以下を一体として扱う考え方です。

- 割引率が上がるほど、クーポン利用率はどう変化するか
- 割引率ごとに、1件あたりの粗利益がどう変化するか
- その両方を掛け合わせたとき、どの割引率帯で期待粗利益が最大になるか
- その山を基準として、実際のクーポン推移をどう設計するか

つまり、  
**「どの割引率がいちばん使われやすいか」ではなく、  
「どの割引率帯が、事業として最も粗利益を取りやすいか」**  
を見極めるための仕様です。

---

## 2. 仕様

本仕様では、割引率ごとのクーポン利用率を指数関数で近似し、  
その利用率と粗利益単価を掛け合わせることで、期待粗利益を評価します。

### 2-1. 採用する利用率モデル

割引率を `d` とすると、利用率 `u(d)` は次の式で表します。

\[
u(d)=0.0100 \cdot e^{4.14d}
\]

ここで、

- `d` = 割引率（例: 0.10 = 10%）
- `u(d)` = 想定クーポン利用率

この式は、  
「割引率毎のクーポン利用率定義」シートをもとに、  
割引率上昇に対する利用率の増え方を近似したものです。

### 2-2. 粗利益単価の考え方

商品単価を `P`、原価率を `c`、割引率を `d` とすると、  
1件あたりの粗利益 `g(d)` は次の式で表します。

\[
g(d)=P(1-d-c)
\]

ここで、

- `P` = 商品単価
- `c` = 原価率
- `d` = 割引率

つまり、

- 割引率が上がると販売単価は下がる
- 原価は一定比率でかかる
- その差額が粗利益になる

という、ごく基本的な構造です。

### 2-3. 期待粗利益

期待粗利益 `G(d)` は、  
「利用率」と「1件あたり粗利益」を掛け合わせて求めます。

\[
G(d)=P(1-d-c) \cdot 0.0100 \cdot e^{4.14d}
\]

この式によって、  
「高い割引率のほうが使われやすいが、値引きしすぎると1件あたり利益が減る」  
という綱引きを、1本の式で評価できます。

### 2-4. 最適割引率

この期待粗利益を最大にする割引率を求めると、最適点 `d*` は次の式になります。

\[
d^* = 1 - c - \frac{1}{4.14}
\]

これは、原価率 `c` が決まれば、  
最大粗利益となる理想割引率の中心点を計算できるという意味です。

---

## 3. 詳細な計算式

ここでは、各式が何を意味しているかを分解して記載します。

### 3-1. 利用率モデル

\[
u(d)=a e^{bd}
\]

今回の採用値は、

- `a = 0.0100`
- `b = 4.14`

なので、

\[
u(d)=0.0100 \cdot e^{4.14d}
\]

です。

この式の意味は、

- 割引率が低いときの利用率は低い
- 割引率が上がるほど、利用率は直線ではなく加速的に上がる

ということです。

### 3-2. 売価

割引後の売価 `s(d)` は、

\[
s(d)=P(1-d)
\]

です。

たとえば単価 3600 円の商品に対して、

- 10%引きなら 3240 円
- 20%引きなら 2880 円
- 40%引きなら 2160 円

となります。

### 3-3. 原価

原価 `cost` は、

\[
cost=P \cdot c
\]

です。

たとえば単価 3600 円、原価率 35% なら、

\[
cost=3600 \cdot 0.35=1260
\]

です。

### 3-4. 1件あたり粗利益

したがって、1件あたり粗利益 `g(d)` は、

\[
g(d)=s(d)-cost
\]

\[
g(d)=P(1-d)-Pc
\]

\[
g(d)=P(1-d-c)
\]

となります。

### 3-5. 期待粗利益

期待粗利益 `G(d)` は、

\[
G(d)=u(d) \cdot g(d)
\]

なので、

\[
G(d)=0.0100 \cdot e^{4.14d} \cdot P(1-d-c)
\]

です。

これが本仕様の中心式です。

### 3-6. 最適割引率の意味

期待粗利益の山になる割引率は、

\[
d^* = 1 - c - \frac{1}{4.14}
\]

です。

今回、原価率を `0.35` とすると、

\[
d^* = 1 - 0.35 - \frac{1}{4.14}
\]

\[
d^* \approx 1 - 0.35 - 0.2415
\]

\[
d^* \approx 0.4085
\]

したがって、  
**理論上の最大粗利益点は 約 40.9% 引き** です。

## 4. サンプルのPHPコード

```php
<?php

declare(strict_types=1);

const HAYA_TOKU_USAGE_MODEL_A = 0.0100;
const HAYA_TOKU_USAGE_MODEL_B = 4.14;

/**
 * 割引率から利用率を推定する
 *
 * $discountRate: 0.10 = 10%
 */
function estimateCouponUsageRate(float $discountRate): float
{
    $d = max(0.0, min(1.0, $discountRate));
    $usageRate = HAYA_TOKU_USAGE_MODEL_A * exp(HAYA_TOKU_USAGE_MODEL_B * $d);

    return max(0.0, min(1.0, $usageRate));
}

/**
 * 割引率から1件あたり粗利益を計算する
 *
 * $unitPrice: 商品単価
 * $costRate: 原価率（0.35 = 35%）
 * $discountRate: 割引率（0.10 = 10%）
 */
function calcUnitGrossProfit(
    float $unitPrice,
    float $costRate,
    float $discountRate
): float {
    $d = max(0.0, min(1.0, $discountRate));
    $c = max(0.0, min(1.0, $costRate));

    return $unitPrice * (1.0 - $d - $c);
}

/**
 * 割引率ごとの期待粗利益を計算する
 */
function calcExpectedGrossProfit(
    float $unitPrice,
    float $costRate,
    float $discountRate
): array {
    $usageRate = estimateCouponUsageRate($discountRate);
    $unitGrossProfit = calcUnitGrossProfit($unitPrice, $costRate, $discountRate);
    $expectedGrossProfit = $usageRate * $unitGrossProfit;

    return [
        'discount_rate' => $discountRate,
        'usage_rate' => $usageRate,
        'unit_gross_profit' => $unitGrossProfit,
        'expected_gross_profit' => $expectedGrossProfit,
    ];
}

/**
 * 指定範囲内で最大粗利益となる割引率を探す
 */
function findBestGrossProfitDiscountRate(
    float $unitPrice,
    float $costRate,
    float $minDiscountRate = 0.10,
    float $maxDiscountRate = 0.60,
    float $step = 0.001
): array {
    $best = null;

    for ($d = $minDiscountRate; $d <= $maxDiscountRate + 1e-9; $d += $step) {
        $row = calcExpectedGrossProfit($unitPrice, $costRate, $d);

        if ($best === null || $row['expected_gross_profit'] > $best['expected_gross_profit']) {
            $best = $row;
        }
    }

    return $best;
}
```

## 5. 実際の使用例（具体的クーポン推移）

ここでは、次の条件で考えます。

- 商品単価：3600 円
- 原価率：0.35
- 最大粗利益点：およそ 40.9%

この場合、  
クーポン推移は **40% 前後を中心に設計する** のが基本になります。

### 5-1. 具体的な意味

このロジックが意味するのは、

- 10% 引きでは利益単価は高いが、利用率が低すぎる
- 50% 引きでは利用率は上がるが、利益単価が削られすぎる
- その中間である 40% 前後が、利用率と利益単価のバランスがもっともよい

ということです。

### 5-2. 具体的な推移例

たとえば 6 日間のクーポン推移を作るなら、次のような形が考えられます。

| 日数 | 割引率 | 意図 |
|---|---:|---|
| 1日目 | 50% | 初動の強い訴求 |
| 2日目 | 46% | まだ高訴求を維持 |
| 3日目 | 42% | 最大粗利益点へ近づく |
| 4日目 | 40% | 最大粗利益帯 |
| 5日目 | 36% | 利益確保寄りへ移行 |
| 6日目 | 32% | 終盤の利益重視 |

このようにすると、

- 前半で強い魅力を出し
- 中盤で最大粗利益帯を通り
- 後半で利益確保寄りに移る

という設計になります。

### 5-3. より山を意識した例

もう少し「最大粗利益点 41% を山」として意識するなら、次のようにもできます。

| 段階 | 割引率 |
|---|---:|
| 開始時 | 48% |
| 初期減衰 | 45% |
| 中盤1 | 42% |
| 中盤2 | 41% |
| 中盤3 | 39% |
| 後半 | 35% |
| 終盤 | 30% |

この考え方では、  
**41% 付近を単なる通過点ではなく、中心帯として長めに使う** ことができます。

### 5-4. 実務上の見方

この仕様は、「常に 41% 固定で出す」という意味ではありません。

そうではなく、

- どの帯を中心にクーポン推移を組むか
- どこを最も厚く使うべきか
- 初動訴求と利益確保のバランスをどこで取るか

を決めるための **基準軸** です。

つまり、

- 高すぎる割引は利益を削りすぎる
- 低すぎる割引は利用率が立ち上がらない
- その中間の **最大粗利益帯** を中心に推移を設計する

というのが、本仕様の核心です。

## 6. この仕様書が表すもの

この仕様書が表しているのは、  
単なる数式や単なる PHP サンプルではありません。

これは、

- 利用率の増え方
- 粗利益の減り方
- その両方を踏まえた最適帯
- その最適帯を中心に組む具体的なクーポン推移

を一つにまとめた、  
**クーポン推移の最大粗利益設計ルール** です。

したがって、今後 HAYA-TOKU では、  
まずこの考え方を **マックスクーポン推移の原本** として扱い、  
必要に応じて実績データで係数調整していく方針とします。
